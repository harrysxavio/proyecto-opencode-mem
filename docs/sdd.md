# SDD

Spec-Driven Development organizes larger work into phases: init, explore, propose, spec, design, tasks, apply, verify, archive, and onboarding when needed.

## Template contract

Every SDD template returns `SUBAGENT_RESULT`. `sdd-init` also returns `SDD_INIT_PACKET`.

## Runtime boundary

The templates are not copies of the live runtime. They are sanitized starter prompts.
